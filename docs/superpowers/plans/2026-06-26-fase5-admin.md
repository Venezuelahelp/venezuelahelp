# VenezuelaHelp — Fase 5: Admin / Backoffice (Implementation Plan)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`). SPA components follow `frontend-public/DESIGN.md` (institutional tokens, reused).

**Goal:** Un backoffice protegido (Cognito) para que el patrocinador edite la config del bot, active/desactive fuentes, dispare un scrape manual y vea un dashboard de conteos. Admin API detrás de un authorizer JWT de Cognito; SPA estática en S3+CloudFront.

**Architecture:** API Gateway HTTP API + authorizer JWT (Cognito User Pool) → Lambda `admin-api` (router que reutiliza `ConfigRepo`/`SourceRepo`/`ItemRepo` de Fase 1 e invoca el Lambda del scraper). SPA Vite+React (`frontend-admin/`) con login vía Amplify Auth, configurada por un `config.json` en runtime (pool/client/apiUrl) que despliega el `AdminStack`. Hosting S3+CloudFront (OAC).

**Tech Stack:** Node 20, `@aws-sdk/client-lambda`, Zod; Vite+React+TS, `aws-amplify` v6 (Auth); CDK Cognito + HTTP API + JWT authorizer + S3/CloudFront.

## Global Constraints

- TypeScript strict; alias `@/`. Sin `console.log` (Powertools en backend).
- Reutiliza Fase 1 (`@/shared/repos/*`, `@/shared/types`, `@/shared/logger`). NO modifica su comportamiento.
- Admin SPA reusa los tokens OKLCH de `frontend-public/DESIGN.md` (mismo sistema institucional). Mobile-friendly; accesible.
- Cognito: sin self sign-up (el admin crea usuarios por CLI). Flujo `USER_PASSWORD_AUTH`.
- API: todas las rutas protegidas por JWT authorizer. CORS: bearer token (no cookies) → `allowOrigins:["*"]` aceptable, `allowHeaders:["authorization","content-type"]`.
- Conventional Commits con emoji; rama `feat/fase5-admin`.
- Deploy: por el problema SSO del SDK de CDK, usar `eval "$(aws configure export-credentials --profile VenezuelaHelp --format env)"` + `CDK_DEFAULT_ACCOUNT=720115910277`. El BucketDeployment requiere `frontend-admin/dist` (build antes del synth).
- **Alcance MVP:** config + sources(toggle+scrape) + dashboard(stats). FUERA: logs Q&A, moderación de ítems (fast-follow).

### Task 1: admin-api router

**Files:** `backend/src/admin-api/router.ts`, `backend/src/admin-api/__tests__/router.test.ts`. Modify `backend/package.json` (add `@aws-sdk/client-lambda`).

**Interfaces:**

- Consumes: `ConfigRepo`, `SourceRepo`, `ItemRepo` (`@/shared/repos/*`), `CATEGORIES` (`@/shared/types`), `@aws-sdk/client-lambda` (`InvokeCommand`).
- Produces: `interface RouteDeps { configRepo; sourceRepo; itemRepo; invokeScraper: () => Promise<void> }` and `route(method: string, path: string, body: unknown, deps: RouteDeps): Promise<{ status: number; body: unknown }>` handling:
  - `GET /config` → `configRepo.get()`.
  - `PUT /config` → validate body with Zod (`scrapeRateMin:number int 5..1440`, `bedrockModelId:string`, `systemPrompt:string`, `botTriggerMode: "mention"|"command"|"all"`) → `configRepo.put(parsed)` → `{status:200, body:parsed}`; invalid → `{status:400,...}`.
  - `GET /sources` → `sourceRepo.list()`.
  - `PATCH /sources/{id}` → body `{enabled:boolean}` → `sourceRepo.get(id)`; if null → 404; else `put({...src, enabled})` → 200 updated.
  - `POST /scrape` → `invokeScraper()` → `{status:202, body:{started:true}}`.
  - `GET /stats` → `{ counts: Record<category, number> (via itemRepo.listByCategory length), sources: (await sourceRepo.list()).map(s=>({id,nombre,enabled,lastRun,lastStatus})) }`.
  - unknown → `{status:404, body:{error:"not found"}}`.
